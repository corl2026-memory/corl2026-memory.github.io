About the Workshop

Memory is central to embodied intelligence.

A robot acting in the physical world cannot treat every timestep as a fresh start — it may need to remember that a mug was placed in a cupboard, that a drawer was already checked, or that a grasp failed because an object slipped. Yet despite rapid progress in vision–language–action (VLA) models, most systems still struggle to store past experience and retrieve it when it matters.

This workshop focuses on what robots should remember — and how memory should be represented, retrieved, integrated with policies, and evaluated.

Open Problems

Open problems we aim to address

01

Defining memory for robots

Which distinctions — short-term context, recurrent state, episodic, semantic, spatial, skill memory, experience replay — actually help build better agents, and which are merely inherited from human cognition?

02

Identifying what to store

Raw observations, language descriptions, object states, maps, action traces, failed attempts, or compressed latents — and how should this differ across manipulation, navigation, and human–robot interaction?

03

Consolidation & forgetting

How should an agent compress repeated experience into reusable knowledge while preserving rare but critical events — and what are principled mechanisms for forgetting stale or misleading memories?

04

Retrieval under partial observability

How can a robot retrieve task-relevant memories when the current observation is insufficient — indexing by objects, places, goals, instructions, or causal relevance — without surfacing distracting or stale entries?

05

Integration with policies & VLAs

Should memory be an external module, an internal latent state, a structured world model, or part of an end-to-end policy — and how can it serve both fast reactive control and longer-term reasoning?

06

Learning from failures

What should a robot remember about a failed grasp, search, or misunderstood instruction — and how can such memories be transformed into improved behavior rather than merely retrieved?

07

Benchmarks for genuine memory

What partially observable, long-horizon tasks isolate genuine memory use from larger context windows or environment memorization — and what metrics capture retrieval quality, recovery, and persistence?

Call for Papers

Contribute to the conversation

  • Topics. Memory architectures, VLA models, long-horizon manipulation, embodied agents, planning, world models, human–robot interaction, and benchmark design.
  • Formats. Short papers and extended abstracts — works-in-progress and negative/ablation results are explicitly welcome.
  • Review criteria. Technical correctness, reproducibility, and actionable artifacts — protocols, datasets, metrics, or well-documented system case studies.
  • Presentation. Posters with optional lightning pitches; we expect roughly 30 contributed posters / short papers.
  • Accepted abstracts (and optional PDFs, with author permission) will be hosted here so attendees can find relevant work.
